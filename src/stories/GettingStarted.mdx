**Tier 1 (10/10):**

- `<AspectRatio>` — 7 ratio presets + numeric ratio
- `<Avatar>` + `<AvatarGroup>` — src / initials / icon fallback, +N overflow

- `<Empty>` — empty-state shell with icon / title / description / action slots
- `<Kbd>` + `<KbdGroup>` — keyboard shortcut display
- `<Label>` — form label with `required` indicator
- `<Separator>` — horizontal / vertical divider, optional label
- `<Skeleton>` — text / circular / rectangular / rounded, pulse / wave / none
- `<Spinner>` — loading indicator with sr-only fallback
- `<Typography>` — h1-h4, p, lead, large, small, muted, blockquote, list, code, kbd; RTL-safe

**Tier 2 (11/12):**

- `<Button>` + `buttonVariants()` helper — 5 variants (solid / soft / outline / ghost / link), 5 sizes, 3 shapes, icon / loading / active
- `<ButtonGroup>` + `<ButtonGroupSeparator>` + `<ButtonGroupText>` — fused buttons via CSS sibling selectors, orientation, context-aware separator
- `<Link>` — underline hover / always / none, 5 sizes, `external` auto handling, `buttonVariants()` for anchor-as-button
- `<Input>` — outline / soft / ghost variants, 5 sizes, 3 shapes, `aria-invalid`, file: modifier
- `<InputGroup>` + `<InputGroupInput>` + `<InputGroupTextarea>` + `<InputGroupAddon>` + `<InputGroupButton>` + `<InputGroupText>` — shadcn-style single-wrapper border, `align` prop (inline-start / inline-end / block-start / block-end)
- `<Textarea>` + `autoGrow` + `maxRows` — scrollHeight-based line clamp
- `<Switch>` — native `role="switch"`, controlled / uncontrolled, 5 sizes
- `<Checkbox>` — `indeterminate` 3rd state via DOM property + `useEffect`
- `<CheckboxGroup>` + `<CheckboxGroupItem>` — multi-select, `<fieldset>` + `<legend>`, context-driven
- `<Radio>` — native `type="radio"`, mutually-exclusive grouping via shared `name`
- `<RadioGroup>` + `<RadioGroupItem>` — single-select, `role="radiogroup"`, auto `name` (useId fallback)

## Shared vocabulary

**Variants** (visual hierarchy): `solid` (filled high-contrast) · `soft` (filled muted) · `outline` (bordered) · `ghost` (transparent) · `link` (inline text)

**Sizes**: `xs` · `sm` · `md` · `lg` · `xl`

**Shapes** (where applicable): `square` · `rounded` · `pill` · `circle`

**Palette:** zinc-only — `text-zinc-{50→900}`, `bg-white`, `bg-zinc-{50→900}`, `border-zinc-{200→900}`. No color schemes (primary / danger / success etc.); theming is handled separately.

## Quick examples

### Form composition

```tsx
import { Label, Input, Typography, Button } from "eglador-ui-react";

<div className="flex flex-col gap-1.5">
  <Label htmlFor="email" required>Email</Label>
  <Input id="email" type="email" placeholder="you@example.com" />
  <Typography variant="muted">Used to verify your account.</Typography>
</div>

<Button>Submit</Button>
```

### Search bar (InputGroup)

```tsx
import {
  InputGroup,
  InputGroupInput,
  InputGroupAddon,
  InputGroupButton,
} from "eglador-ui-react";
import { Search } from "lucide-react";

<InputGroup>
  <InputGroupInput placeholder="Search…" />
  <InputGroupAddon align="inline-start"><Search /></InputGroupAddon>
  <InputGroupAddon align="inline-end">
    <InputGroupButton variant="solid" size="sm">Search</InputGroupButton>
  </InputGroupAddon>
</InputGroup>
```

### Single-select (RadioGroup)

```tsx
import { RadioGroup, RadioGroupItem } from "eglador-ui-react";

<RadioGroup
  legend="Plan"
  name="plan"
  defaultValue="pro"
  onValueChange={(v) => console.log(v)}
>
  <RadioGroupItem value="free" label="Free" description="Solo projects" />
  <RadioGroupItem value="pro" label="Pro" description="For professionals" />
  <RadioGroupItem value="enterprise" label="Enterprise" description="Custom" />
</RadioGroup>
```

### Anchor styled as a button

```tsx
import { buttonVariants } from "eglador-ui-react";

<a href="/docs" className={buttonVariants({ variant: "outline", size: "sm" })}>
  Documentation
</a>
```

## Design principles

- **Headless-first** — native HTML semantics are the foundation; styling is opt-in via Tailwind classes
- **Compound API** — Group / Item / Addon parts wired via context
- **A11y** — `role`, `aria-*`, focus-visible ring, `<fieldset>` / `<legend>` / `<label htmlFor>`
- **TypeScript-first** — generic props, strict mode
- **Tailwind v4** — `zinc` palette, `rounded-sm` default, logical CSS (`ps-*`, `me-*`, RTL automatic)
- **Zero runtime deps** — `clsx` and `tailwind-merge` are pre-bundled
- **Controlled + Uncontrolled** — every stateful form component supports both patterns
